# SOL Lens provenance

This document makes the Build Week extension boundary explicit.

## Pre-existing foundation

Before OpenAI Build Week, the project already had the **SOL Engine** concept and its foundational semantic mathematics. That prior work includes the ideas of atomic semantic units (Logons), evidence density, pressure, coherence/flow, and governance constraints.

No claim is made here that those foundational ideas were created during the hackathon window.

## New Build Week work

Development of **SOL Lens** began July 18, 2026 as a new application of that foundation to GPT-5.6 agent migration. The following work is new:

- Developer Tools product concept: a semantic trace and migration workbench
- GPT-5.5 baseline versus GPT-5.6 candidate workflow
- observable-trace-only product contract
- typed Logon graph fixture and dependency edges
- interactive filtering and Logon inspection
- deterministic scoring profile and promotion court
- JSON proof-packet schema and export flow
- responsive Vinext/React application
- Cosmic Semantic Lab × Solar Atlas visual system
- Build Week documentation, tests, and deployment

## Phase 2 packet-driven extension

- Date: July 19, 2026
- Scope: canonical v0.2 packet ingestion, v0.1 migration, strict validation,
  deterministic cycle-aware layout, scalable graph modes, pan and zoom,
  overview grouping, local evaluation comparison, and lossless graph export
- Status: implemented and verified locally before any replacement deployment

This extension reuses the existing SOL scoring profile without changing the
foundational mathematics. Screen coordinates and overview groups are view
concerns; imported metrics and verdicts are treated as claims to compare with
the locally replayed result.

## Development assistance

The new SOL Lens application was designed and implemented collaboratively with OpenAI Codex and the GPT-5.6 family. The application code, visual system, scoring fixture, testing, and documentation were produced during the Build Week extension window.

## Evidence policy

SOL Lens evaluates observable inputs and outputs only. It does not claim access to private chain-of-thought or hidden model reasoning. A production integration should retain raw observable events or stable references to them, version its scoring profile, and preserve the model identifier returned by the API.
